# TranscriptSource boundary + TranscriptTailSource + encodeCwd + transcript->stream-json normalizer and noise-filter

## Objective

Build the structured-data recovery path for the interactive substrate behind a clean Shannon-swap seam, with NO PTY dependency so it is fully unit-testable. This is the single riskiest production component in IDEA-013: it must absorb the proven transcript-vs-wire schema divergence (Probe E recorded 45%+ of real interactive-transcript lines drop to `{ kind: '__unknown__' }` against `streamParser/schemas.ts`), win the spawn→first-`.jsonl` discovery race, and survive the `encodeCwd` collision case — all without coupling to `@homebridge/node-pty-prebuilt-multiarch` or the `@anthropic-ai/claude-agent-sdk`. The deliverable is four new production files under a new `main/src/services/panels/claude/transcript/` directory: (1) `transcriptSource.ts` — the narrow `TranscriptSource` interface (the seam: no pty/SDK/narrowing imports); (2) `encodeCwd.ts` — the standalone `/`,`.`,non-ASCII→`-` algorithm verified against the live `~/.claude/projects/` layout, with the #19972 collision note; (3) `transcriptNormalizer.ts` — the MANDATORY transcript-shape→stream-json-shape normalizer + noise-filter that is the single place the schema divergence is absorbed (so `narrow()` and the structured Claude panel stay unchanged); (4) `transcriptTailSource.ts` — the roll-our-own `TranscriptSource` implementation (discovery → inode+offset tail → normalize → forward to `onLine`, with turn-end markers on `onTurnEnd`). This task does NOT spawn a PTY, does NOT call `narrow()`/`emit('output')`, and does NOT touch the manager, the factory, `index.ts`, `runExecutor.ts`, or any IDEA-029 file — those are S3 (TASK consuming this) and the depends-on-MERGE slices.

## Implementation Steps

1. **Create the directory + `main/src/services/panels/claude/transcript/encodeCwd.ts`** (new). Export `export function encodeCwd(absPath: string): string`. Algorithm (verified against the live `~/.claude/projects/` entries, e.g. `/private/var/folders/…/T/cyboflow-day3-19dmtv/.cyboflow-worktrees/prune-f1c214bb` → `-private-var-folders-…-T-cyboflow-day3-19dmtv--cyboflow-worktrees-prune-f1c214bb`): replace every character that is not `[A-Za-z0-9_]` — including `/`, `\`, `.`, and any non-ASCII char — with a single `-`. A leading separator therefore yields a leading `-`, and adjacent separators (e.g. `/` immediately followed by `.`) yield `--` (confirmed: `0` directory names on disk contain a literal dot). Add a comment block documenting the GitHub #19972 collision caveat (distinct abs paths can collide on one key) — the `TranscriptTailSource` cwd-binding fallback (step 7) is the mitigation. No `any`; pure function, no imports.

2. **Create `main/src/services/panels/claude/transcript/transcriptSource.ts`** (new) — the SEAM. Define `export interface TranscriptSource { start(onLine: (obj: <NormalizedPanelObject>) => void): Promise<void>; stop(): void; waitForFirstLine(timeoutMs: number): Promise<void>; }`. Declare a small local result/discriminant type for the normalizer's output if needed, but keep this module FREE of `@homebridge/node-pty-*`, `@anthropic-ai/claude-agent-sdk`, the `streamParser` barrel, and `TypedEventNarrowing` (narrowing happens later in the manager, per the Shannon-swap rationale in the synthesis Architecture section). The `onLine` callback receives ALREADY-NORMALIZED objects (stream-json-shaped); the interface does not itself import the normalizer impl. Add an `onTurnEnd?` callback hook to the start signature (or a separate setter) so the tail can surface turn-end markers for S3 completion without forwarding them as panel events.

3. **Create `main/src/services/panels/claude/transcript/transcriptNormalizer.ts`** (new) — the MANDATORY normalizer (folds Probe E). Export a pure function, e.g. `export function normalizeTranscriptLine(parsed: unknown): NormalizeResult`, where `NormalizeResult` is a small discriminated union: `{ kind: 'panel'; event: unknown }` (a stream-json-shaped object to forward to `onLine`), `{ kind: 'turn-end'; marker: 'stop_hook_summary' | 'turn_duration' }`, or `{ kind: 'drop' }` (or return `null` for drop — pick one and keep it consistent). Logic:
   - **Drop set (noise):** top-level `type` ∈ { `last-prompt`, `mode`, `permission-mode`, `bridge-session`, `attachment`, `ai-title`, `file-history-snapshot`, `queue-operation` } → `{ kind: 'drop' }`; and `type === 'system'` with `subtype` ∈ { `local_command`, `bridge_status` } → `{ kind: 'drop' }`. These never reach `narrow()` so `raw_events` is not bloated with `__unknown__` rows.
   - **Turn-end side channel:** `type === 'system'` with `subtype` ∈ { `stop_hook_summary`, `turn_duration` } → `{ kind: 'turn-end', marker: subtype }`. Do NOT forward as a panel envelope (`schemas.ts` does not model these subtypes — confirmed: `systemUnionSchema` (schemas.ts:146-152) covers only `init`/`compact_boundary`/`hook_started`/`hook_response`/`status`).
   - **Panel-critical mapping:** `assistant` lines (text/thinking/tool_use content) → map into the `assistantEventSchema` shape (schemas.ts:164-184: `{ type:'assistant', message:{ id, model, role:'assistant', content:[…] }, … }`). `user` lines whose `message.content` is an ARRAY of tool_result blocks → map into `userEventSchema` (schemas.ts:190-207). `user` lines whose `message.content` is a STRING (which FAIL `userEventSchema`'s `z.array(toolResultBlockSchema)` requirement — Probe E) → wrap the string into the accepted array/text shape (or route to a documented side path) so it is NOT dropped. Preserve `session_id`/`sessionId` and `cwd` where the manager will need them, but DO NOT depend on `system/init` (it never appears interactively).
   - Fail-soft: any line that is neither drop, turn-end, nor a mappable panel line returns `{ kind: 'drop' }` (or passes through as `panel` only if it already matches a schema variant) — never throw. No `any`; use `unknown` + narrowing guards (e.g. `isRecord(v): v is Record<string, unknown>`).

4. **Create `main/src/services/panels/claude/transcript/transcriptTailSource.ts`** (new) implementing `TranscriptSource`. Imports: `fs`, `path`, `os` from node; `{ encodeCwd }` from `./encodeCwd`; `{ normalizeTranscriptLine }` from `./transcriptNormalizer`; the `TranscriptSource` interface from `./transcriptSource`; and a structural logger type (`Pick<Logger, 'warn' | 'error' | 'verbose'>` from `../../../../utils/logger`, mirroring how `claudeCodeManager.ts` imports `type { Logger }`). Constructor takes `(opts: { worktreePath: string; projectsRoot?: string; discoveryTimeoutMs: number; logger: <structural logger> })` — `projectsRoot` defaults to `path.join(os.homedir(), '.claude', 'projects')` but is injectable so tests use a temp dir (hermetic; never touches the real `~/.claude`). Do NOT omit the logger (CLAUDE.md optional-logger rule) — all diagnostics gate on it.

5. **Discovery (`waitForFirstLine` + `start`):**
   - On `start`, snapshot the set of existing `*.jsonl` basenames in `<projectsRoot>/<encodeCwd(worktreePath)>/`.
   - Watch for a NEW `*.jsonl` via `fs.watch` on that dir with a `setInterval` poll fallback (fs.watch is unreliable on some platforms). When a new basename appears that is NOT in the snapshot, treat the basename (sans `.jsonl`) as the candidate session UUID.
   - Bound the wait by `discoveryTimeoutMs`. On timeout, call `this.logger.error('[Cyboflow Transcript] discovery timeout …')` (LOUD) and reject `waitForFirstLine` (or resolve a failure discriminant the manager finalizes as 'failed') — never an unbounded await.

6. **Tail loop:** once a file is bound, tail it line-buffered:
   - Track `inode` (from `fs.stat`) and a byte `offset`. Read appended bytes from `offset`; accumulate into a rolling buffer; split on `\n` (mirror the framing in `cyboflowMcpServer.ts:69-79` — `recvBuffer += chunk; while ((nl = recvBuffer.indexOf('\n')) !== -1) { line = recvBuffer.slice(0, nl).trim(); recvBuffer = recvBuffer.slice(nl+1); … }`). A line split across two writes is reassembled by the rolling buffer.
   - Re-sync on truncation/append: if `fs.stat` shows a new inode OR size < offset, reset offset to 0 (file rotated/truncated).
   - For each complete line: `JSON.parse` inside try/catch — on parse failure `this.logger.warn(…)` and `continue` (fail-soft; never crash). Pass the parsed object to `normalizeTranscriptLine`. If `kind==='panel'` → call `onLine(result.event)`. If `kind==='turn-end'` → call the `onTurnEnd` hook with the marker. If `kind==='drop'` → skip.
   - `stop()` clears the watcher, the poll interval, and the tail interval/stat loop so the loop exits and no watchers leak.

7. **Collision fallback (corrected key):** track a `bound` flag. Before binding the session, if more than one candidate file is plausible (collision), do NOT bind on mtime alone — read each candidate's early lines and bind only to the file whose first line bearing a TOP-LEVEL `cwd` equals the target worktree abs path. Per the synthesis: the literal first physical line is `file-history-snapshot` (no `cwd`), so binding waits for the first cwd-bearing line; do NOT reference `system/init.cwd` (never appears interactively). Use the camelCase top-level `sessionId` only as a secondary cross-check against the filename basename.

8. **Create `main/src/services/panels/claude/transcript/__tests__/encodeCwd.test.ts`** — assert the three encodeCwd cases (slash path, dot-segment `--` collapse matching the live entry, non-ASCII→hyphen) and that the collision comment exists. Plain vitest unit test, no fs.

9. **Create `main/src/services/panels/claude/transcript/__tests__/transcriptNormalizer.test.ts`** — feed golden fixture lines mirroring the Probe-E inventory (each noise type, each system subtype, assistant text/thinking/tool_use, user array-content, user STRING-content, stop_hook_summary, turn_duration). Assert: every noise/system-noise line → drop; turn-end markers → turn-end discriminant; panel-critical lines normalize to objects that `new TypedEventNarrowing(spyLogger).narrow(result.event)` returns as a typed variant (NOT `kind==='__unknown__'`); STRING-content user line is not dropped and survives `narrow()`. (Importing `TypedEventNarrowing` is fine in the TEST — only the production seam files must stay narrowing-free.)

10. **Create `main/src/services/panels/claude/transcript/__tests__/transcriptTailSource.test.ts`** — use `os.tmpdir()` for a hermetic projects root and a `makeSpyLogger()` returning `{ warn: vi.fn(), error: vi.fn(), verbose: vi.fn() }`. Cover: (a) discovery of a newly-written `*.jsonl` basename as the session UUID within an injected `discoveryTimeoutMs`; (b) discovery-timeout against an empty dir → reject + spy logger called; (c) incremental tail of a real fixture written in chunks including a split line and a malformed line → exactly the valid normalized panel objects forwarded in order, malformed skipped, no throw; (d) `onTurnEnd` fires on a stop_hook_summary/turn_duration line; (e) truncation+re-append (rewrite the file) is survived (inode/offset re-sync); (f) `stop()` exits the loop / leaks no watchers; (g) collision: two new files in the same key dir with different top-level `cwd` → binds the cwd-matching one. `afterEach` calls `source.stop()` and removes temp dirs.

11. Run `pnpm test:unit` and confirm the three new files pass. If a `better-sqlite3` NODE_MODULE_VERSION error appears (it should not — these files do not touch the DB), run `pnpm rebuild better-sqlite3` per CLAUDE.md and re-run. Then `pnpm typecheck && pnpm lint` clean.

- **The normalizer is the SINGLE absorption point.** Per Probe E + Q3 (synthesis), `schemas.ts` is the stream-json WIRE schema and the on-disk transcript is a DISTINCT schema; `narrow()` (typedEventNarrowing.ts) and the panel are intentionally left unchanged. The normalizer must NOT call `narrow()` itself — it only RESHAPES; narrowing is the manager's job in S3. The `any`-free grep allows `unknown` + guards for parsing the transcript line.
- **Turn-end markers are SECONDARY for completion** (Probe C: Stop shell hook is PRIMARY in S3; `stop_hook_summary`/`turn_duration` are the SECONDARY observed markers). This task only surfaces them on the side channel; it does NOT implement completion (that is S3).
- **No `system/init` reliance.** `systemInitSchema` exists in schemas.ts (61-80) but Probe B/E confirmed `system/init` NEVER appears interactively — discovery and cwd-binding must use the filename basename (UUID) + the first top-level-`cwd`-bearing line, never `system/init`.
- **The TranscriptSource interface is a hard seam.** The grep AC enforces zero node-pty/SDK/narrowing imports in `transcriptSource.ts` (the Shannon swap point: a future `ShannonTranscriptSource` is a one-factory-branch swap per the Shannon Decision). The TAIL implementation may import `fs`/`path`/`os` and the normalizer, but still must NOT import the narrowing layer (grep AC on transcriptTailSource.ts).
- **Logger must be passed, not omitted** (CLAUDE.md). `TranscriptTailSource` takes a required structural logger and gates discovery-timeout/malformed/watch-fallback diagnostics on it; the timeout test asserts the spy was actually called.
- **Real fixtures.** Per the slice, goldens are derived from a live no-`-p` capture (recorded in `docs/probes/IDEA-013-probe-findings.md` from S0). If a live capture is unavailable in the verifier environment, the fixtures must mirror the exact Probe-E line shapes recorded in that findings doc — not invented shapes.
