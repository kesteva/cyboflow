# Probe-first kill-criteria gate: empirically resolve the load-bearing unknowns before any production code

## Objective

Run, BEFORE any IDEA-013 production slice, a fixed battery of empirical probes (A, A2, B, C, D, E, F, G, H) against a live, installed interactive `claude` binary and a real Pro/Max subscription, and write the results to a one-page decision record at `docs/probes/IDEA-013-probe-findings.md` that resolves the four open questions (Q1 substrate choice, Q2 interactive-hook gating, Q3 panel preservation, Q4 completion detection) and names, for every downstream slice (S1-S7), the explicit fallback its gating probe selects. This is a pure investigation task: it ships NO production code. Throwaway probe artifacts — a ~10-line `.claude/settings.json` PreToolUse hook and `probe-transcript.ts` — live under `docs/probes/scratch/` and are never wired into the app build or `package.json`. The SDK substrate keeps shipping unchanged while this runs, so blocking the epic on these probes is zero-cost. Three probes are designed to collapse work before it starts: Probe A can redirect S5 to the native-TUI gating fallback, Probe E confirms S2 MUST ship a normalizer (not a bare tail), and Probe H is a whole-epic go/no-go business-risk gate that ends in an explicit USER sign-off (human-needed) on parallel-interactive use against a real plan. NOTE: this planning session already ran Probe E and parts of B/C empirically and they changed the design — those results are folded forward in the record and MUST be re-confirmed against the exact `--settings`-isolated, no-`-p` spawn config used by the future InteractiveClaudeManager. This task has `depends_on: []`: it is independent of IDEA-029 and is NOT a depends-on-MERGE task (the depends-on-MERGE serialization applies to S4/S5/S6, downstream of this probe).

## Implementation Steps

1. **Create the scratch worktree + probe artifacts under `docs/probes/scratch/`.** This keeps the throwaway hook config and script out of the app's `tsconfig`/`asarUnpack`/`.claude/` so they cannot affect `pnpm dev` or a packaged build. Create:
   - `docs/probes/scratch/settings.json` — a ~10-line `.claude/settings.json` PreToolUse `*` hook entry whose command runs the throwaway hook script; set the hook `timeout` field HIGH (several minutes) to probe the multi-minute block. To actually exercise it, copy this file to the probe worktree's real `.claude/settings.json` for the duration of the probe (do NOT commit it into the project's live `.claude/settings.json`). Mirror the wire framing the real shell hook will use later: read PreToolUse JSON on stdin, write `{hookSpecificOutput:{hookEventName:'PreToolUse',permissionDecision:'deny'}}` to stdout, exit 2.
   - `docs/probes/scratch/probe-transcript.ts` — a throwaway Node/TS script that (a) feeds a real captured interactive transcript JSONL line-by-line through `claudeStreamEventSchema.safeParse` (imported read-only from `main/src/services/streamParser/schemas.ts`) and counts `{kind:'__unknown__'}` results + tabulates the unmodeled top-level `type`s and `system` subtypes (Probe E), and (b) verifies the `encodeCwd` slash/non-ASCII->hyphen algorithm against a live `~/.claude/projects/<key>/` directory name and measures the spawn->first-`.jsonl`-write delay (Probe B). No `any` — use `unknown` + narrowing for parsed lines.

2. **Probe A (do FIRST among probes — gates whether a GATED interactive substrate ships at all, S5 PRIMARY).** With the scratch `.claude/settings.json` in place and `CYBOFLOW_ORCH_SOCKET` exported in the env, launch interactive `claude` (NO `-p`) against the installed version, prompt it to make a Bash call from the MAIN session, and record PASS/FAIL for each of the five sub-claims: (a) the hook fires for the MAIN session's own Bash call; (b) exit-2 + the JSON `permissionDecision:'deny'` BLOCKS synchronously; (c) the hook subprocess can block for MINUTES (sleep inside the hook) without `claude` killing it — distinct from a 5-10s timeout; (d) the hook subprocess inherits `CYBOFLOW_ORCH_SOCKET` from the PTY env (echo it from inside the hook); (e) the exit-2-vs-JSON-`permissionDecision` precedence in interactive (non-SDK) mode. Write the verdict and, if ANY sub-claim fails, record the named KILL: S5 takes the native-TUI fallback (no roll-our-own gating) and the substrate picker (S7) surfaces 'approval routing unavailable'.

3. **Probe A2 (subagent gating + AskUserQuestion — ship-gating).** In the same worktree under the Probe-A hook, have the agent spawn a Task subagent and attempt a Bash call; record whether the `.claude/settings.json` hook FIRES for the subagent's tool call. Record the KILL: if it does NOT fire, ungated subagent tool calls are a SHIP BLOCKER for subagent-spawning workflows (planner/sprint/compound) on the interactive substrate, so S5/S7 must restrict interactive selection for those workflows OR force-deny the Task tool — not merely document it. Separately attempt an AskUserQuestion and confirm there is NO field that injects the user's selected answers through a command hook (`updatedInput` is SDK-`HookJSONOutput`-only); record AskUserQuestion as a native-TUI-only v1 limit (no QuestionRouter wiring on this substrate).

4. **Probe B (session-id discovery + encodeCwd + early cwd line — gates S1/S2).** Run interactive `claude` in a known absolute-path worktree. Using `probe-transcript.ts`: (a) verify `encodeCwd('/Users/x/Developer/...')` matches the live `~/.claude/projects/<key>/` directory name and record a concrete example plus a non-ASCII->hyphen case + the #19972 collision note; (b) measure the spawn->first-`.jsonl`-write delay and record a concrete `DISCOVERY_TIMEOUT_MS` value (with margin); (c) confirm the session UUID is ONLY in the filename (`--session-id` ignored interactively, #44607); (d) confirm a top-level `cwd` field is present on an EARLY transcript line and that the literal first physical line is `file-history-snapshot` with NO `cwd`, so S2 collision disambiguation binds on the first cwd-bearing line (NOT `system/init.cwd`, which never appears interactively).

5. **Probe C (re-scoped — how a no-`-p` turn terminates the PTY, gates S3 completion).** On a FRESHLY captured no-`-p` interactive run (NOT a cyboflow/bridge-session transcript — those carry extra line types), confirm there is NO terminal `{type:'result'}` line and the REPL does NOT self-exit (returns to the prompt). Confirm the turn-end mechanism: whether a Stop event shell hook fires (PRIMARY), and that `system/stop_hook_summary` + `system/turn_duration` lines appear at turn end (SECONDARY). Pair with a hung-input case to confirm the chosen signal distinguishes a finished turn from one waiting on input. Record this as the completion-signal selection for S3 (EOF/`/exit` to PTY stdin on the turn-end signal; inherited PTY onExit as the teardown CONSEQUENCE).

6. **Probe D (Shannon bridge status — gates Q1).** Re-read Shannon `GOAL_PROGRESS.md` at planning time; confirm the bidirectional permission-gating bridge is still 'Planned, implementation not started' and that the Bun+tmux runtime deps remain unacceptable for the 2026-06-15 deadline. Record the Q1 resolution: ROLL-OUR-OWN now, behind the swappable `TranscriptSource` seam (a future `ShannonTranscriptSource` is a one-factory-branch swap).

7. **Probe E (HARD GATE — transcript-vs-wire schema parity, gates S2 normalizer).** Run `probe-transcript.ts` against a real interactive transcript captured on the EXACT `--settings`-isolated, no-`-p` spawn config the future InteractiveClaudeManager will use. Record the re-measured `{kind:'__unknown__'}` rate (planning observed 45%+), the unmodeled top-level inventory (last-prompt / mode / permission-mode / bridge-session / attachment / ai-title / file-history-snapshot / queue-operation) and `system` subtypes (stop_hook_summary / turn_duration / local_command / bridge_status / api_error), the STRING-content `user` line failure against `userEventSchema`'s array requirement, and the camelCase top-level `sessionId` / never-appearing `system/init`. Commit the CONCLUSION: a normalizer + noise-filter is MANDATORY for S2 (not optional drift patching).

8. **Probe F (interactive MCP load + report_step fires — note; gated behind IDEA-029 socket up).** When IDEA-029's socket runtime is up under `pnpm dev`, launch interactive `claude` with `--mcp-config` injecting the cyboflow stdio entry and confirm `cyboflow_report_step` is listed/callable from the MAIN session AND that a PROMPT-BODY-PREPENDED step-reporting instruction actually CAUSES a `report_step` call (watch `cyboflow-backend-debug.log` for `handleReportStep`). If IDEA-029 is not yet merged at probe time, record this as a deferred sub-probe to run against the merged tree, with the fallback for S6 (per-worktree instruction file if the prepend proves unreliable).

9. **Probe G (socket round-trip incl. human-decision window — de-risks S5, after A passes + socket up).** Measure the ApprovalRequest->verdict round-trip through the shell hook INCLUDING a multi-minute human-decision window via the heartbeat; confirm `claude` does not kill the hook subprocess and the held-open socket is not evicted by TASK-798's connection lifecycle. Record the fallback: if `claude` kills the hook before a human can respond, or the socket is evicted mid-wait, S5 falls to the Probe-A-fail native-TUI path.

10. **Probe H (NEW, ship-gating business risk — whole-epic go/no-go).** Run N>=4 parallel PTY-driven interactive `claude` sessions against a real Pro/Max plan; record whether all complete or hit rate-limit / throttle / concurrency caps, any usage-limit responses, and per-session allowance burn. Cite the PRIMARY support article `support.claude.com/articles/15036540` (blesses interactive terminal/IDE use but is SILENT on automated/parallel/headless driving). ESCALATE to the user for an explicit business-risk sign-off (human-needed) and record their decision in the findings doc — OR record an explicit decision to ship the value prop as an UNCONFIRMED assumption. This is the one AC that requires a human, not an automated check.

11. **Write `docs/probes/IDEA-013-probe-findings.md`** as a one-page decision record: a Q1-Q4 resolution table (each citing the probe evidence) and a per-slice fallback table naming, for S1-S7, the fallback each gating probe selects. Keep it concise (one page).

12. **Confirm the probe artifacts do not regress the build.** Verify `git diff --stat` shows ZERO changes under `main/src`, `frontend/src`, `shared/`, or `package.json`, then run `pnpm test:unit && pnpm typecheck && pnpm lint` and confirm all exit 0 (the scratch artifacts under `docs/probes/scratch/` are excluded from the app `tsconfig`/build). If a `better-sqlite3` NODE_MODULE_VERSION error appears, run `pnpm rebuild better-sqlite3` first per CLAUDE.md.

## Acceptance Criteria notes

- **This is an investigation task, not a code task.** The only durable deliverable is the decision record; the `settings.json` + `probe-transcript.ts` artifacts are throwaway evidence-gatherers. They are deliberately placed under `docs/probes/scratch/` (NOT in the live `.claude/` or `scripts/`) so they are outside the app build and cannot break `pnpm dev` or packaging. The `git diff --stat` AC enforces the no-production-code boundary.
- **Probe H ends in a human gate.** The "explicit user business-risk sign-off" cannot be satisfied by an automated assertion — it requires the USER to review the parallel-interactive ToS/concurrency evidence (and the SILENT-on-automation support article) and record a go/no-go decision, OR an explicit recorded decision to ship the "8 parallel agents on your subscription" value prop as an UNCONFIRMED assumption. The verifier checks that the record contains a dated user-decision line, not that any test passed.
- **Probe A is the master gate for S5 and for whether a GATED interactive substrate ships at all.** The five sub-claims (fires for MAIN session, blocks synchronously, blocks for MINUTES, inherits `CYBOFLOW_ORCH_SOCKET`, exit-2-vs-JSON precedence) each get an explicit PASS/FAIL; any FAIL routes S5 to native-TUI gating. This mirrors the framing the future shell hook (`preToolUseShellHook.ts`, S5) and `cyboflowMcpServer.ts:64-90,126-127` wire framing will rely on.
- **Probe E re-measures on the EXACT spawn config**, not the planning-session sample, because the planning transcripts were cyboflow/bridge-session-driven and may carry different line types. The re-measured unknown-rate and unmodeled-type inventory are the empirical basis for the S2 mandatory-normalizer AC.
- **The `any`-free grep AC applies to `probe-transcript.ts`** even though it is throwaway, because it is in-repo TypeScript that imports the production `claudeStreamEventSchema`; use `unknown` + narrowing for parsed transcript lines (CLAUDE.md `no-any` rule, `@typescript-eslint/no-explicit-any` = error).
- **`pnpm test:unit` (NOT `pnpm test:e2e`)** is the standing verifier gate per CLAUDE.md; `test:e2e` is environmental/non-functional in headless verifiers. The gate here only proves the probe artifacts add no regressing production path.
- **No depends-on-MERGE framing applies to THIS task.** `depends_on: []` — S0-probe is independent of IDEA-029 and runs first. The depends-on-MERGE serialization (branch off the merged IDEA-029 tree for index.ts / claudeCodeManager.ts / runExecutor.ts / mcpQueryHandler.ts) is a constraint on the DOWNSTREAM slices S4/S5/S6, which this record documents but does not perform.
